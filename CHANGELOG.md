# 更新日志 (Changelog)

本文档记录了"基金助手"(Fund Helper) VS Code 扩展的所有重要更改。

## [0.2.7] - 2026-05-06

### 优化改进 (Improvements)

- 💡 **TreeView 信息完整化**：
  - 基金列表项展开详情和 Tooltip 现在显示完整的四个字段：**估算涨幅、估算收益、当日涨幅、当日收益**
  - 每个字段后面显示对应的日期标注（估算：当日日期，当日：上一交易日日期）
  - 顺序与表格视图统一，提升信息一致性
- 💄 **信息排序优化**：TreeView Tooltip 和详情项现在按照表格视图的顺序显示数据，使用户体验更一致
- 🐛 **修复 TreeView 数据显示不一致**：修复基金列表项中，标签和实际收益计算不匹配的问题，现在准确区分估算收益和当日收益

### 技术优化 (Technical)

- ♻️ **TreeView 重构**：重新组织 TreeView Tooltip 和详情项的显示顺序和格式，确保与表格视图数据一致

## [0.2.6] - 2026-05-06

### 问题修复 (Bug Fixes)

- 🐛 **修复交易日开盘前数据显示**：修复交易日开盘前若 API 已返回数据时，表头日期显示为上一个交易日的问题
- 🐛 **改进开盘前收益判断逻辑**：修改为根据 API 数据是否已更新到今天来判断是否显示估算收益，而不是仅根据交易时间段判断

### 优化改进 (Improvements)

- 💡 **智能数据更新判断**：
  - 若 API 已返回今天的数据（gztime 已更新到当日），则显示最新的估算涨幅和收益，即使未到交易时间
  - 若 API 数据仍为昨天，则显示"--"并显示当今日期，等待数据刷新
  - 休市日期间始终显示"休市"和"--"（灰色显示）
- 💄 **用户体验优化**：表头日期更准确，能即时反映 API 是否有最新数据，减少用户困惑

### 技术优化 (Technical)

- ♻️ **数据检查函数**：添加 `isGzTimeUpdatedToday()` 函数，检查 gztime 是否已更新到当日
- ♻️ **判断逻辑升级**：改进 `generateFundRow()` 和 `calculateEstimatedGain()` 中的逻辑，从时间段判断改为数据更新状态判断
- ♻️ **表头日期动态显示**：`getEstimatedDate()` 现在根据数据是否已更新来动态显示对应的日期

## [0.2.5] - 2026-05-06

### 问题修复 (Bug Fixes)

- 🐛 **改进开盘前收益判断逻辑**：修改为根据 API 数据是否已更新到今天来判断是否显示估算收益，而不是仅根据交易时间段判断

### 优化改进 (Improvements)

- 💡 **市场状态判断优化**：增强市场状态判断，从仅判断"是否休市"改为区分三种状态：休市（灰色显示 `休市`）、未开盘（显示今天日期）、已开盘（显示实时数据）
- 💄 **开盘前交互优化**：未开盘时，估算涨幅和估算收益列显示灰色"--"，下方显示当天日期而不是过时的时间，提升用户体验

### 技术优化 (Technical)

- ♻️ **市场状态传递优化**：在市场状态消息中添加 `todayDate` 字段，允许 webview 准确显示当前日期，无需依赖已过时的 API 数据
- ♻️ **日期显示逻辑**：改进 `getEstimatedDate()` 函数，智能判断当前状态显示对应的日期信息

## [0.2.4] - 2026-04-30

### 问题修复 (Bug Fixes)

- 🐛 **修复数据源问题**：修复 `fundgz` 接口返回的 `navchgrt` 可能是旧数据的问题，现在始终优先使用 `MNFInfo` 接口的最新数据
- 🐛 **修复 StatusBar 显示问题**：修复 StatusBar 在休市时仍显示上一个交易日真实收益的问题，现在改为显示估算收益（盘中数据）
- 🐛 **修复日收益计算**：统一 StatusBar 和 Webview 的日收益计算公式，确保两个视图显示一致的数据
- 🐛 **修复休市逻辑**：StatusBar 在休市时显示"基金统计（休市）"，并提示"当前为休市时间，估算数据不可用"

### 优化改进 (Improvements)

- 💄 **StatusBar 显示优化**：
  - 交易日：显示"今日基金统计"，包含上涨/下跌基金数量、估算收益、估算收益率
  - 休市日：显示"基金统计（休市）"，只显示持有金额和累计收益信息
- 💄 **数据一致性**：确保 StatusBar、Webview 表格视图、TreeView 三个视图的数据计算逻辑完全一致

### 技术优化 (Technical)

- ♻️ **数据合并优化**：在 `src/fundService.ts` 中优化数据合并逻辑，始终优先使用 MNFInfo 的最新数据（`navchgrt`、`jzrq`、`dwjz`）
- ♻️ **计算公式统一**：将 `calcDailyGain` 函数简化为与 Webview 一致的计算方式：`(持有金额 × 涨跌幅) / 100`
- ♻️ **StatusBar 重构**：重构 StatusBar 的统计逻辑，从"当日收益"改为"估算收益"，并添加休市判断

## [0.2.3] - 2026-04-30

### 新增功能 (Features)

- ✨ **分组统计增强**：分组标签 hover 显示详细统计 Tooltip，包含基金数量、估算收益/涨幅、当日收益/涨幅、持有收益/收益率、总资产/成本等 9 项统计信息
- ✨ **分组统计复制**：分组 Tooltip 新增复制按钮，一键复制分组统计信息到剪贴板
- ✨ **隐私模式增强**：隐私模式下 hover 敏感数据可临时查看，鼠标移开自动恢复模糊，添加 `cursor: help` 提示
- ✨ **休市状态显示**：非交易日（周末、节假日）时，估算涨幅和估算收益显示为 `--`，估算日期显示为 `休市`，避免用户误解过时数据

### 优化改进 (Improvements)

- 💄 **估算涨幅列优化**：估算涨幅列第二行改为显示更新时分（HH:mm 格式，如 14:03），方便用户了解数据新鲜度，休市时显示 `休市`
- 💄 **固定列阴影优化**：基金名称列（固定列）在表格横向滚动时动态显示右侧阴影，视觉层次更清晰，提升滚动体验
- 💄 **日期标注优化**：
  - 估算涨幅/估算收益：表头和 Tooltip 显示估算日期（如 `04-30`），休市时显示 `休市`
  - 当日涨幅/当日收益：表头和 Tooltip 显示净值日期（上一个交易日，如 `04-29`）
  - 分组统计：Tooltip 中估算和当日数据都添加日期标注
- 💄 **加权平均计算**：分组统计使用持有金额加权计算平均涨幅，更准确反映分组整体表现
- 💄 **休市收益显示**：非交易日显示最后一个交易日的真实收益，而不是显示为 0，保持数据连续性
- 💄 **颜色标识统一**：分组 Tooltip 中的收益和涨幅使用红涨绿跌颜色标识，与基金 Tooltip 风格一致
- 💄 **行情中心优化**：优化行情中心页面样式和布局，支持媒体查询，小屏幕下观感更佳，提升数据展示清晰度
- 💄 **基金详情页优化**：改进基金详情页的信息展示和操作体验，风格统一，信息层次更清晰
- 💄 **QDII 基金支持**：优化 QDII 基金的净值计算逻辑，避免部分 QDII 基金因缺少实时估值导致显示异常的问题

### 技术优化 (Technical)

- ♻️ **代码重构**：优化 `renderGroupTags()` 函数，计算更详细的分组统计信息
- ♻️ **样式优化**：新增 `.group-tooltip` 样式，与 `.fund-tooltip` 保持一致的视觉风格；优化固定列阴影样式，添加平滑过渡动画
- ♻️ **事件处理优化**：优化分组标签 hover 事件处理，支持快速切换分组标签时正确显示 Tooltip，避免误关闭；新增表格滚动监听，动态控制固定列阴影显示
- ♻️ **隐私模式样式**：为隐私模式添加 hover 交互效果，包含 `transition` 动画和 `cursor` 提示

### 问题修复 (Bug Fixes)

- 🐛 **休市逻辑修复**：修复非交易日仍显示过时估算数据的问题，现在明确显示 `休市` 状态
- 🐛 **日期显示修复**：修复表头日期显示逻辑，估算日期和净值日期分别显示，不再混淆
- 🐛 **分组 Tooltip 修复**：修复快速切换分组标签时 Tooltip 被错误关闭的问题，提升交互体验
