分组管理里面分为上下两个模块，分别是分组管理器（拖拽排序 / 编辑 / 删除），支持新增、修改、删除分组，未分类的分组不能修改和删除，其实未分类就是一个用来存放未分类的基金的一个预设分组。分组item可以拖拽排序，当然了未分类的分组也不支持拖拽排序。分组item拖拽排序会影响分组item在外面的显示顺序。点击分组管理器的分组item，下面的模块会出现基金列表，显示的是在该分组下的基金列表，用户可以调节基金列表里面的顺序，用来决定在该分组下基金的排列，也可以拖拽基金列表里面的基金到分组管理器里面的分组，来实现分组的修改。需要你将web端的分组管理进行全面优化，弹窗使用我们分组管理的弹窗就像参照图那样进行实现。


紧接着是基金列表，基金列表里面包含多种数据，具体你可以看webview的实现，基金名称是两行组成，第一行是基金的名称，支持ellipsis，第二行是基金代码和基金所属的分组tag。需要注意的是基金拖拽在不同分组下的表现不一样，如果在全部分组中，那么只有在默认排序时才能拖拽，如果是在其他分组中，同样也是在默认排序时才能够进行拖拽排序，这个排序会影响基金在分组中的顺序。

基金列表，其实是一个table，那么th下面有一些日期，估算收益和估算涨幅下面的日期是最新一个交易日的日期，而其他列除了关联板块外没有日期，其他列th的第二行日期是上一个交易日的日期。
