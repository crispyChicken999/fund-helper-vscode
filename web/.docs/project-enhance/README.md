## 推荐施工顺序

```
Phase 0 → Phase 1（优先数据真实化+列表完整化）
        → Phase 2（行情中心，相对独立）
        → Phase 3（详情页）
        → Phase 4（数据同步，功能完整后再做）
        → Phase 5（UI精化，最后做）
```

> **注意**：Phase 1 的「基金数据主流程」是整个项目的地基，必须最先完成，后续所有列的显示都依赖它。

## 关键技术依赖

| 需求 | 推荐库 |
|------|-------|
| 拖拽排序（分组/列） | `sortablejs` + `@vueuse/integrations` |
| 长按手势 | 自定义 composable（无需额外库） |
| 二维码扫描 | `html5-qrcode`（轻量） |
| ECharts 图表 | CDN 引入（避免打包膨胀） |
| 随机 ID 生成 | `crypto.getRandomValues`（原生，无需 nanoid） |
| 滑动关闭弹窗 | `@vueuse/core` useDraggable |
